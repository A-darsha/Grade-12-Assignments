Layers of Data Security

Data security involves multiple layers to protect data from unauthorized access and breaches. Here are the key layers with examples:

  • Physical Security: Protects the physical infrastructure. Example: Locked server rooms with biometric access control.
  • Network Security: Protects data transmitted over networks. Example: Using firewalls and VPNs to secure data in transit.
  • Database Security: Protects data stored in databases. Example: Implementing encryption and access control lists (ACLs) to restrict database access.
  • Application Security: Ensures software applications are secure. Example: Regularly updating software to patch vulnerabilities.
  • Operational Security: Policies and procedures for handling data. Example: Regular security training for employees.

Data Integrity

Data integrity ensures data remains accurate and reliable. Key aspects include:

  • Entity Integrity: Ensures each table has a unique primary key. Example:
StudentID StudentName
1 John Doe
2 Jane Smith
  • Referential Integrity: Ensures foreign keys match primary keys. Example:
CourseID StudentID CourseName
101 1 Mathematics
102 2 English

The StudentID in the `Course` table must match an existing StudentID in the `Student` table.

  • Domain Integrity: Ensures data values conform to a defined format. Example:
EmployeeID EmployeeName Email
1 Alice Johnson alice.johnson@example.com
2 Bob Smith bob.smith@example.com

The Email field must follow the email format (e.g., name@domain.com).

  • Consistency: Ensures data consistency across systems. Example:

If an employee's address is updated in one system, it should be updated in all related systems to maintain consistency.

Anomalies

Data anomalies are inconsistencies that can occur in databases. Common types include:

  • Insertion Anomaly: Occurs when certain attributes cannot be added without other attributes. Example:
CourseID CourseName Instructor
101 Mathematics Dr. Adams

Inserting a new course might require specifying an instructor, which can be problematic if no instructor is available yet.

  • Update Anomaly: Occurs when updates are not consistently applied. Example:

1 Math Dr. Smith smith@example.com
