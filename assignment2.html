Definition of Keys

In a Database Management System (DBMS), keys are crucial for maintaining the integrity and uniqueness of data. They are used to uniquely identify records and establish relationships between tables. The main types of keys are:

  • Primary Key: A unique identifier for a record in a table.
  • Foreign Key: A field in one table that links to the primary key of another table.
  • Unique Key: Ensures all values in a column are distinct.
  • Composite Key: A primary key consisting of multiple columns.
  • Candidate Key: A set of columns that can uniquely identify a record.
  • Super Key: A set of one or more columns that can uniquely identify a record.
  • Alternate Key: Candidate keys that are not chosen as the primary key.

Primary Key

The Primary Key in a DBMS is a unique identifier for each record in a table. It ensures that each record is unique and can be uniquely identified by its key value. A primary key cannot contain NULL values and must be unique across the table. It is crucial for maintaining the integrity of the data and is often used to establish relationships between tables.

Example of Primary Key
StudentID StudentName Age
1 Alice 20
2 Bob 22

Foreign Key

A Foreign Key is a field (or collection of fields) in one table that uniquely identifies a row of another table. It is used to establish and enforce a link between the data in the two tables. Foreign keys ensure referential integrity by making sure that the value in the foreign key field matches one of the values in the primary key field of the referenced table.

Example of Foreign Key
OrderID ProductID (Foreign Key) Quantity
101 1 5
102 2 10

Unique Key

A Unique Key constraint ensures that all values in a column (or a set of columns) are distinct from each other. Unlike the primary key, a unique key can contain NULL values, but all non-null values must be unique. It is used to enforce uniqueness on columns that are not the primary key but still require distinct values.

Example of Unique Key
EmployeeID Email (Unique Key) Department
001 alice@example.com HR
002 bob@example.com Finance

Composite Key

A Composite Key (or Compound Key) is a primary key that consists of two or more columns. These columns, when combined, uniquely identify each record in a table. Composite keys are used when a single column is not sufficient to uniquely identify a record, and multiple columns are needed to ensure uniqueness.

Example of Composite Key
OrderID ProductID Quantity
101 2001 5
102 2002 10

Candidate Key

A Candidate Key is a set of one or more columns that can uniquely identify a row in a table. Each candidate key is a potential primary key, and the database designer must choose one as the primary key. All candidate keys are unique and can be used to enforce uniqueness on the table's rows.

Example of Candidate Key
StudentID (Candidate Key) StudentEmail (Candidate Key) StudentName
1 alice@example.com Alice
2 bob@example.com Bob

Super Key

A Super Key is a set of one or more columns that can uniquely identify a record in a table. A super key can contain additional attributes that are not necessary for unique identification. It is a superset of candidate keys, meaning that it includes one or more candidate keys.

Example of Super Key
StudentID StudentEmail StudentName DateOfBirth
1 alice@example.com Alice 2004-05-15
2 bob@example.com Bob 2003-11-20

Alternate Key

An Alternate Key is any candidate key that is not selected as the primary key. These keys still have the potential to uniquely identify records in a table but are not chosen as the primary key.

Example of Alternate Key
EmployeeID (Primary Key) EmployeeEmail (Alternate Key) EmployeeName
101 alice@example.com Alice
102 bob@example.com Bob
